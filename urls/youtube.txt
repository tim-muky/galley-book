# YouTube recipe URLs — structured test batch
# One URL per line; lines starting with # are ignored (used here as section headers).
#
# Run: TEST_SESSION_COOKIE="..." npx tsx scripts/test-import.ts urls/youtube.txt
#
# Categories below are documented so failures can be analysed by type instead of
# only by overall pass/fail counts. When adding URLs, drop them under the right
# section so future test runs keep apples-to-apples coverage.
#
# Expected outcome legend (set in test-set notes, not parsed by the runner):
#   PERFECT  = name + image + ingredients + steps + prep_time
#   GOOD     = name + ingredients + steps (image/prep optional)
#   PARTIAL  = name only (no ingredients/steps) — usually the source had no recipe text
#   FAILED   = explicit error (e.g. no recipe content available)


# === A. Description-rich — recipe in the video description (should be GOOD/PERFECT) ===

# === B. Captioned cooking videos — recipe via transcript (should be GOOD) ===
# Major recipe channels with auto/manual captions. Tests the youtube_transcript route.


# === C. No description, no captions — recipe via Gemini video analysis (best-effort) ===
# Hardest case. Expect PARTIAL or GOOD, never PERFECT.

# to /watch?v=ID before downstream calls (see canonicalYouTubeUrl in route.ts).
